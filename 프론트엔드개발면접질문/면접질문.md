JavaScript

1. Event Loop에 대해서 설명하시오.

넵, 이벤트 루프는 JavaScript의 싱글 스레드 기반의 환경에서 비동기 작업을 처리하기 위한 실행 메커니즘입니다.
JavaScript는 한 번에 하나의 작업만 처리하지만, 이벤트 루프를 통해 논 블로킹(non-blocking) 방식의 비동기 처리가 가능합니다.

이벤트 루프에는 4가지의 구조로 되어있습니다.
첫 번째, Call Stack은 JavaScript가 실행되는 곳입니다. LIFO 구조 방식으로 작동하며 동기 코드가 먼저 실행됩니다.
두 번째, Macrotask, Microtask Queue가 Web API, Node API의 백그라운드로 넘어가 처리가 된 후 Callback Queue에서 임시 대기합니다.
세 번째, Call Stack이 비었다면 모든 Microtask Queue가 Call Stack으로 넘어가 실행됩니다.
네 번째, Microtask Queue가 비었다면 하나의 Macrotask Queue가 Call Stack으로 넘어가 실행 됩니다. 이후 위 과정을 반복합니다.

2. 호이스팅에 대해서 설명하시오.

넵, 호이스팅은 JavaScript에서 코드를 실행하기 전, 선언된 변수/함수를 범위내의 스코프 최상단으로 끌여올려 처리하는 동작 방식을 얘기합니다.
변수 선언방식은 var, let, const 가 있습니다. 해당 방식은 호이스팅되는 방식 또한 다른데
첫 번째, var는 선언과 초기화가 한 번에 이뤄지고 undefined를 출력합니다.
두 번째, let/const는 선언만 호이스팅 되며, 할당은 호이스팅이 되지 않아 ReferenceError를 출력합니다.
세 번째, 함수 또한 호이스팅이 되는데 선언과 정의 모두 호이스팅되어 어디서든 호출이 가능합니다.
네 번째, 변수를 통한 함수 호출을 했을 때 변수만 호이스팅 되고 함수 정의 호이스팅은 되지 않습니다. 함수는 할당 이후에만 사용 가능합니다.
