Callback Function

JavaScript의 콜백 함수 개념과 사용법, 그리고 Promise와 async/await로의 발전 과정을 이해합니다.

콜백 함수란 무엇인가?

콜백 함수(Callback Function)는 다른 함수의 인자로 전달되어 특정 시점에 실행되는 함수입니다. JavaScript에서 비동기 작업을 처리하는 기본적인 방법 중 하나입니다.

콜백 함수의 기본 개념

콜백 함수는 다음과 같은 특징을 가집니다:

  • 다른 함수의 인자로 전달됩니다.
  • 특정 조건이나 이벤트가 발생했을 때 실행됩니다.
  • 비동기 작업의 완료를 처리하는 데 주로 사용됩니다.

콜백 함수 예제

// 기본 콜백 함수 예제
function greet(name, callback) {
  console.log('Hello, ' + name);
  callback();
}

function sayGoodbye() {
  console.log('Goodbye!');
}

greet('World', sayGoodbye);

// 결과:
// Hello, World
// Goodbye!

비동기 콜백 함수

콜백 함수는 주로 비동기 작업에서 사용됩니다:

// setTimeout 예제
setTimeout(function() {
  console.log('이 메시지는 1초 후에 출력됩니다.');
}, 1000);

// 이벤트 리스너 예제
button.addEventListener('click', function() {
  console.log('버튼이 클릭되었습니다.');
});

// AJAX 요청 예제
fetch('/api/data')
  .then(function(response) {
    return response.json();
  })
  .then(function(data) {
    console.log(data);
  });

콜백 지옥(Callback Hell)

콜백 함수를 중첩해서 사용하면 코드가 복잡해지고 가독성이 떨어집니다:

// 콜백 지옥 예제
getData(function(a) {
  getMoreData(a, function(b) {
    getMoreData(b, function(c) {
      getMoreData(c, function(d) {
        // 코드가 계속 중첩됨...
      });
    });
  });
});

Promise와 async/await

콜백 지옥을 해결하기 위해 Promise와 async/await가 도입되었습니다:

// Promise 사용
getData()
  .then(a => getMoreData(a))
  .then(b => getMoreData(b))
  .then(c => getMoreData(c))
  .then(d => {
    // 깔끔한 체이닝
  });

// async/await 사용
async function fetchData() {
  const a = await getData();
  const b = await getMoreData(a);
  const c = await getMoreData(b);
  const d = await getMoreData(c);
  // 동기 코드처럼 보이는 비동기 처리
}

실무에서의 활용

콜백 함수는 여전히 많은 곳에서 사용됩니다:

  • 이벤트 리스너
  • 배열 메서드 (map, filter, forEach 등)
  • 타이머 함수 (setTimeout, setInterval)
  • Node.js의 비동기 작업

콜백 함수는 JavaScript의 비동기 프로그래밍의 기초입니다. Promise와 async/await가 등장했지만, 콜백 함수는 여전히 많은 상황에서 유용하게 사용됩니다. 콜백 지옥을 피하기 위해 Promise나 async/await를 적절히 활용하는 것이 좋습니다.
